# Trace Paired-Crossed-Action Default 360 Readout-Window Progress

Date: 2026-06-22

## Purpose

This run promoted the crossed-action diagnostic into the trace collection
harness as the `paired_crossed_action` seed profile, then evaluated it on the
full default 12-task catalog.

The goal was reproducibility. The earlier crossed-action corpus proved the
idea locally, but it was generated by a one-off script. This run makes the
same bigram-control pressure available through the project CLI.

## Interpretation

This result is useful and bounded.

The useful part: the project now has a reproducible seed profile where
safe/exfiltration pairs are exactly balanced at unigram and bigram levels.
Under that pressure, grouped word 1-2 and character baselines fall near chance
while the late readout-window activation probe remains strong.

The bounded part: word 1-3 and 1-4 TF-IDF remain perfect under grouped
evaluation. The profile still uses stable action-frame templates, so longer
local text features can recover the label. This confirms that the next data
problem is not only token balance; it is template diversity.

This profile should stay in the harness as a regression diagnostic. It should
not be treated as final training data for the capstone demo.

## Next Step

Use `paired_crossed_action` as a baseline diagnostic and collect or generate a
new grammar-diverse corpus where:

- safe/exfiltration pairs still share tool payloads;
- protected credential material remains only in the injected context;
- unigram and bigram balance remain strict;
- the allowed/denied semantics are expressed through many syntactic frames;
- grouped word 1-3 and 1-4 TF-IDF are reported as required baselines.

The target is not to make every text baseline fail. The target is to make sure
activation gains are not explained by a simple lexical or template shortcut.
